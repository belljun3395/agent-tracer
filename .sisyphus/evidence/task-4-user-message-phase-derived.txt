Task 4 Evidence: Remove runtime-name branching from user-message and lifecycle logic
====================================================================================

Date: 2026-03-17

## Changes Made

### packages/server/src/application/monitor-service.ts

1. Removed `automaticSources` array and conditional sessionId fallback logic.
   - Before: sessionId was derived from latest session for non-automatic sources
   - After: `const sessionId = input.sessionId;` — sessionId is required by type contract

2. Removed `if (input.phase === "initial")` block that updated task title and cliSource.
   - No longer writes cliSource from logUserMessage
   - No longer auto-updates task title from user message

3. Added phase derivation when `input.phase` is omitted:
   ```typescript
   const phase = input.phase
     ?? (this.database.countRawUserMessages(input.taskId) === 0 ? "initial" : "follow_up");
   ```

4. Updated metadata to always include `phase` (not conditional on input.phase being set).

5. Removed `isGenericTaskTitle` private function (now unused).

6. Updated JSDoc on `logUserMessage()` to: "캐노니컬 user.message 이벤트를 기록한다. sessionId는 필수."

### packages/server/src/infrastructure/monitor-database.ts

Added `countRawUserMessages(taskId: string): number` method that queries:
```sql
select count(*) as count
from timeline_events
where task_id = @taskId
  and kind = 'user.message'
  and json_extract(metadata_json, '$.captureMode') = 'raw'
```

### packages/server/src/application/types.ts

Updated JSDoc on `TaskUserMessageInput.phase`:
- Before: "initial = 작업 항목의 첫 메시지; follow_up = 후속 메시지."
- After: "initial = 작업 항목의 첫 메시지; follow_up = 후속 메시지. 생략 시 서버가 기존 raw user.message 이벤트 수로 자동 도출."
