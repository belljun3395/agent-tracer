# ADR-0002 — 번들링하지 않고 소스를 실행한다

## 상태

채택됨

## 결정

서버 애플리케이션과 워커는 TypeScript 소스를 그대로 실행한다. 실행 시점의 트랜스파일러가
타입을 지우고 모듈을 로드한다. 배포 산출물을 만들기 위한 번들러를 쓰지 않는다.

예외는 둘이다. 웹 대시보드는 브라우저로 가야 하므로 번들한다. 에이전트 플러그인의 훅은
사용자의 기계에서 의존성 설치 없이 자립 실행되어야 하므로 번들한다.

## 근거

서버 쪽 번들링에서 얻는 것이 없다. 콜드 스타트는 컨테이너 기동 시간에 묻히고, 배포 크기는
문제가 된 적이 없다. 반면 잃는 것이 크다.

의존성 주입 프레임워크는 데코레이터의 메타데이터에 의존하고, 오케스트레이션 엔진은 워크플로
코드를 자기 방식으로 다시 번들한다. 네이티브 바인딩을 가진 브로커 클라이언트도 있다.
번들러는 이 셋을 동시에 만족시키지 못한다. 한쪽을 맞추면 다른 쪽이 조용히 깨지고, 그 실패는
빌드가 아니라 런타임에 드러난다.

소스를 그대로 실행하면 개발과 운영의 실행 경로가 같아진다. 스택 트레이스가 실제 파일을
가리키고, 재현되지 않는 번들 전용 버그가 없다.

## 결과

- 실행 명령은 트랜스파일러를 로더로 등록하고 진입 파일을 직접 가리킨다.
- 컨테이너 이미지는 소스와 `node_modules`를 담는다. 빌드 단계에서 컴파일하지 않는다.
- 개발 서버는 파일 감시로 재시작한다.
- 웹과 플러그인의 번들 설정은 각자 자기 패키지가 소유한다. 다른 패키지가 그 설정을 공유하지
  않는다.
- 플러그인 번들은 공유 커널의 소스를 안으로 인라인한다. 배포된 훅이 `node_modules` 없이
  실행되는 성질을 지킨다.
