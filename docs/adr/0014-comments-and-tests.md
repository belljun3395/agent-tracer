# ADR-0014 — 주석은 외부 계약만 담고 동작은 테스트가 소유한다

## 상태

채택됨

## 결정

주석에 남기는 것은 셋뿐이다.

- 포트의 계약. 여러 구현체가 따라야 하는 역할. 포트마다 한 문장.
- 경계를 넘는 값의 외부 의미. 요청과 응답의 필드가 무엇을 뜻하는지.
- 코드로 표현할 수 없는 외부 제약. 프로토콜, 단위, 플랫폼 한계.

테스트가 소유하는 것은 이렇다.

- 분기와 필터와 반복
- 재시도와 실패 처리
- 경계값과 회귀 사례
- 내부 데이터 변환
- 실행 순서와 파이프라인 단계
- 객체 생성과 의존성 배선

주석은 한글 한 문장으로 쓴다. 코드 식별자와 기술 고유명사는 영어로 쓸 수 있다.
TypeScript는 JSDoc, Python은 docstring을 쓴다. 구현 안의 줄 주석은 그 줄을 이해하는 데
외부 제약이 꼭 필요할 때만 쓴다.

테스트 이름은 한글 행위 문장이다. `describe`는 대상을 쓰고 `it`은 "…한다"를 쓴다.

## 근거

주석과 코드는 함께 변하지 않는다. 코드를 고치면 컴파일러와 테스트가 항의하지만 주석은 침묵한다.
그래서 주석이 설명하는 동작은 시간이 지나면 거짓말이 된다. 거짓말하는 주석은 없는 주석보다 나쁘다.

동작을 주석이 아니라 테스트에 적으면 이 문제가 사라진다. 코드가 바뀌면 테스트가 깨지고,
깨진 테스트는 무시할 수 없다. 테스트 이름이 한글 행위 문장인 이유가 이것이다.
테스트 목록이 그대로 그 모듈의 명세가 된다.

반대로 테스트가 표현할 수 없는 것이 있다. 이 포트를 구현하는 사람이 무엇을 지켜야 하는지,
이 필드가 밀리초인지 초인지, 왜 소켓 권한을 제한해야 하는지는 코드 바깥의 사실이다.
이것만 주석이 갖는다.

한글로 쓰는 이유는 짧기 때문이다. 한 문장으로 계약을 말하려면 모국어가 유리하다.
길어지면 그것은 이미 계약이 아니라 설명이다.

## 결과

- 유스케이스는 예외 없이 인접 테스트를 갖는다. 허용치가 0이다.
- 응용 계층의 테스트는 포트의 인메모리 대역만 쓴다. 기술을 모른다.
- 모델의 테스트는 대역도 쓰지 않는다. 입력과 판정만 있다.
- 어댑터의 테스트는 포트 계약의 준수를 검증한다.
- 인바운드의 테스트는 와이어 파싱과 위임을 검증한다.
- 주석이 세 종류 중 어디에도 해당하지 않으면 지운다. 그 내용이 필요하면 테스트로 옮긴다.
- "이 분기는 A일 때만 탄다" 같은 주석을 쓰고 싶으면, 그 문장을 테스트 이름으로 쓴다.
