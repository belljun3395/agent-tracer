# ADR-0017 — 에이전트의 명세는 도메인이 소유하고 실행 백엔드는 어댑터다

## 상태

채택됨

## 결정

에이전트의 프롬프트, 출력 스키마, 도구 명세는 제품 지식이므로 도메인 모델이 소유한다.
실행 방식은 어댑터다.

슬라이스는 자기 에이전트 포트를 선언하고, 두 어댑터가 그 포트를 구현한다.
하나는 언어 모델 SDK로 직접 실행하고, 하나는 파이썬 그래프 실행기에 위임한다.
두 어댑터가 같은 명세를 읽고 각자의 방언으로 렌더링한다.

언어 모델 공급자 실행기는 앱의 `config/`에 있다. 프롬프트를 넣으면 구조화된 출력을 낸다.
무슨 프롬프트인지는 모른다.

## 근거

프롬프트는 코드가 아니라 제품이다. "어떤 근거를 봐야 하는지", "심각도를 어떻게 매기는지",
"몇 개를 제안해야 하는지"는 도메인 지식이다. 이것이 실행기 안에 있으면 실행기를 하나 더
추가할 때마다 지식이 복사된다. 복사된 지식은 갈라지고, 갈라진 프롬프트는 백엔드마다 다른
결과를 낸다. 그러면 백엔드를 비교하는 실험 자체가 무의미해진다.

명세를 도메인에 두고 어댑터가 렌더링만 하면, 두 백엔드가 같은 지식으로 실행된다.
그때 비로소 두 백엔드의 차이가 실행 방식의 차이가 된다.

출력 스키마도 마찬가지다. 백엔드마다 스키마 방언이 다르지만 스키마의 의미는 하나다.
의미는 도메인이 갖고, 방언 변환은 어댑터가 한다.

## 결과

- 백엔드는 둘이다. 언어 모델 SDK로 직접 실행하는 것과 파이썬 그래프로 실행하는 것.
  두 구현을 나란히 두고 차이를 관찰하는 것이 목적이므로 어느 쪽도 임시가 아니다.
- 어느 백엔드를 쓸지는 잡 입력이 정한다. 잡 입력에 없으면 앱 설정의 기본값을 쓴다.
- 파이썬 쪽은 무상태다. 프롬프트와 스키마와 도구를 워커에게서 받아 실행만 한다.
  파이썬이 자기 프롬프트를 갖지 않는다.
- 도구도 도메인이 소유한다. 두 도메인이 비슷한 도구를 각자 갖는 것은 정상이다.
  공용 도구로 묶지 않는다.
- 에이전트의 실행 궤적은 스텝 단위로 저장한다. 두 백엔드의 궤적이 같은 형태여야 비교가 된다.
