# ADR-0003 — Node 런타임 버전을 하나로 고정한다

## 상태

채택됨

## 결정

모든 패키지가 Node 24를 쓴다. 루트가 엔진 범위를 선언하고, 컨테이너 이미지와 지속적 통합이
같은 버전을 쓴다. 패키지마다 다른 버전을 허용하지 않는다.

## 근거

메시지 브로커 클라이언트가 네이티브 바인딩을 갖는다. 네이티브 모듈은 Node의 메이저 버전마다
ABI가 달라지므로, 설치할 때의 버전과 실행할 때의 버전이 다르면 로드 자체가 실패한다.
이 실패는 코드와 무관한데 테스트가 수십 개씩 무너지는 모습으로 나타나서, 원인을 찾는 데
시간이 든다.

버전을 하나로 고정하면 이 실패 양식이 사라진다. 고정이 주는 이득이 최신 버전을 따라가는
이득보다 크다.

## 결과

- 루트 `package.json`이 엔진 범위를 선언하고 설치 시점에 검증한다.
- 컨테이너 베이스 이미지와 지속적 통합의 Node 버전이 이 범위 안에 있다.
- 셸의 기본 Node가 다른 버전이면 테스트가 대량으로 실패한다. 코드 문제가 아니므로
  버전부터 확인한다.
- 버전을 올릴 때는 네이티브 의존성의 지원 여부를 먼저 확인하고 한 번에 전부 올린다.
