# ADR-0035 — local 프로파일은 에이전트를 구독 인증으로 키 없이 돌린다

## 상태

채택됨

## 결정

local 프로파일에서 세 AI 잡 에이전트는 Anthropic API 키 대신 로그인된 claude CLI의
구독 자격증명으로 실행한다. 이를 위해 두 가지가 갈린다.

- **기본 백엔드가 claude-sdk다.** python 백엔드는 파이썬 실행기가 직접 API를 호출해 키가
  반드시 필요하다. 키 없이 도는 경로는 하위 claude CLI로 실행하는 claude-sdk뿐이다.
- **claude-sdk가 API 키를 주입하지 않는다.** local이면 `ANTHROPIC_API_KEY`를 넣지 않아 하위
  CLI가 자기 자격증명으로 인증한다.

`AGENT_BACKEND` 환경변수나 잡 입력이 백엔드를 명시하면 그것이 프로파일 기본값보다 우선한다.
prd 프로파일은 python 기본과 사용자별 API 키 주입을 그대로 유지한다.

## 근거

두 백엔드를 비교하는 실험은 유료 API를 쓴다. 로컬에서 반복 테스트할 때마다 과금되면 개발
루프의 비용이 커진다. 사용자의 기계에는 이미 로그인된 claude CLI가 있으므로 그 자격증명으로
돌리면 키 설정도 키당 과금도 없앤다.

서버(prd)에는 로그인이 없고, 멀티테넌시와 약관 때문에 사용자별 API 키가 옳다. 로컬은 사용자
자신의 기계에서 자신의 구독을 쓰는 것이라 그 제약이 없다. 그래서 프로파일이 경계다.

## 구현

- 기본 백엔드 결정은 잡을 스탬프하는 tracer-api와 잡을 실행하는 워커가 각자 프로파일을 본다.
  명시 env가 없을 때만 프로파일이 기본값을 정한다.
- claude-sdk 러너는 프로파일이 local이면 키 주입을 끄고, 하위 CLI 프로세스 환경 허용 목록에
  구독 토큰 변수를 통과시킨다.
- host 실행은 사용자 홈의 로그인을, 컨테이너 실행은 주입한 장기 토큰을 쓴다. 토큰은
  `claude setup-token`으로 발급해 환경변수 `CLAUDE_CODE_OAUTH_TOKEN`으로 넘긴다.
- 배포 이미지 그대로 이 경로를 로컬에서 띄우는 명령이 `stack:up:local`이며, 오버라이드 컴포즈가
  프로파일과 백엔드와 토큰을 건다.

## 결과

- local에서는 키 설정 없이 에이전트를 쓴다. 웹은 키 유무로 기능을 막지 않으므로 화면 변경
  없이 동작한다.
- prd 동작은 바뀌지 않는다. 백엔드를 명시하는 선택도 그대로 우선한다.
- 구독 인증으로 실행해도 SDK가 낸 환산 비용은 관측용으로 계속 기록된다. 지갑에서 나가는
  값이 아니다.
