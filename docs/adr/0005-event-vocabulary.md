# ADR-0005 — 이벤트 어휘를 관측 표준에 맞춘다

## 상태

채택됨

## 결정

이벤트의 종류와 속성 이름은 생성형 AI 관측 표준의 규약을 따른다. 표준에 대응하는 이름이 있으면
표준명을 쓰고, 없으면 이 제품의 접두사를 붙인 이름을 쓴다.

이벤트에는 추적 식별자와 구간 식별자가 있다. 이 둘은 페이로드 안이 아니라 이벤트 원장의 열로
승격되어 저장된다.

어휘는 공유 커널이 단독으로 소유한다. 복사본을 두지 않는다.

## 근거

이 시스템이 수집하는 것은 결국 언어 모델 호출과 도구 실행이다. 같은 것을 이미 관측 업계가
표준으로 정의해 두었다. 표준을 따르면 수집한 데이터를 기존 관측 도구로 그대로 내보낼 수 있고,
내보내지 않더라도 필드의 의미를 두고 다투지 않는다.

자체 어휘를 만들면 처음에는 편하지만, 나중에 외부 도구와 연결할 때마다 매핑 계층을 만들게 된다.
그 매핑 계층은 어휘가 늘 때마다 함께 늘어난다.

추적 식별자를 페이로드 안에 두면 조회할 때마다 JSON을 파헤쳐야 하고 인덱스를 걸 수 없다.
경계를 넘는 순간 열로 승격하는 편이 낫다.

## 결과

- 새 이벤트 종류를 만들 때 표준에 대응하는 이름이 있는지 먼저 확인한다.
- 조회 모델 생성기가 이벤트 원장을 소비하면서 관측 파이프라인으로도 내보낼 수 있다. 같은
  데이터의 두 소비자다.
- 어휘가 바뀌면 플러그인의 버전을 함께 올린다. 구버전 플러그인이 보낸 이벤트를 받아주는
  코드를 남기지 않는다.
